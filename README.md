# Лабораторная работа 14  Git Advanced

## Цель
Изучение .gitignore, Git LFS, git filter-repo, Sparse Checkout.

## Что сделано
1. Создан репозиторий с ошибками (логи, секреты, большие файлы в истории)
2. Настроен .gitignore  убраны app.log и secrets.env из отслеживания
3. Настроен Git LFS для .psd и .zip файлов
4. Через git filter-repo удалён secrets.env из всей истории коммитов
5. Через git lfs migrate import переведён big_photo.psd на LFS
6. Выполнен Sparse Checkout  клонирована только часть репозитория

## Контрольные вопросы

**1. Почему git rm --cached не удаляет файл с диска?**
Флаг --cached работает только с индексом (staging area), оставляя рабочую директорию нетронутой.

**2. Зачем нужен .gitattributes?**
Переопределяет настройки Git для конкретных расширений  например, указывает LFS обрабатывать бинарные файлы.

**3. В чём опасность filter-repo?**
Переписывает историю  все хэши меняются, форки и локальные копии коллег становятся несовместимыми.

**4. Почему LFS лучше обычного хранения больших файлов?**
Git хранит полную копию каждого изменения. LFS хранит только указатели в Git, а сами файлы  на отдельном сервере.

## Выводы
- .gitignore не действует на уже отслеживаемые файлы  нужен git rm --cached
- Git LFS заменяет большие файлы указателями (pointer), экономя место в репозитории
- git filter-repo переписывает историю и требует force push
- Sparse Checkout позволяет работать только с нужной частью монорепозитория
